HTML 시작하기

HTML은 웹 페이지의 뼈대입니다. 이 페이지 하나로 기초부터 실무급 고급 주제까지 빠르게 익힙니다.

가장 작은 HTML 문서

index.html
<!doctype html>
<html lang="ko">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>내 첫 웹페이지</title>
  </head>
  <body>
    안녕하세요, 웹!
  </body>
</html>
  • doctype 문서가 HTML5임을 브라우저에 알립니다.
  • lang 기본 언어 설정으로 접근성과 SEO에 중요합니다.
  • viewport 모바일 화면에서 올바른 스케일링을 보장합니다.

태그, 요소, 속성

  • 태그: 예: <p> 열고 </p> 닫습니다.
  • 요소: 시작태그 + 내용 + 종료태그(또는 빈 요소).
  • 속성: <img src="..." alt="..."> 처럼 추가 정보.
  • 빈 요소: <meta>, <br>, <img> 등 종료태그 없음.
ℹ️
문서 한글 설정: <meta charset="utf-8"> 누락 시 한글이 깨질 수 있습니다.

핵심 문법과 기본 구조

블록 vs 인라인

  • 블록 요소: 줄을 바꾸고 가로폭 전체 사용. 예: <div>, <p>, <section>
  • 인라인 요소: 줄내에서 흐름 유지. 예: <span>, <a>, <strong>
예시
<p>문장 중 <strong>강조</strong>와 <a href="#">링크</a>는 인라인입니다.</p>

전역 속성

  • id: 문서 내 고유 식별자.
  • class: 스타일/스크립트 훅.
  • title: 요소 보조 설명(툴팁).
  • hidden: 표시 안 함(접근성 고려 필요).
  • data-*: 사용자 정의 데이터 저장.
예시
<button id="buy" class="btn primary" title="구매하기">구매</button>
<div data-user-id="42">사용자</div>
문서 머리 정보(head) 필수 메타와 예시

메타와 링크

<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <meta name="description" content="페이지 설명">
  <link rel="icon" href="/favicon.ico">
  <title>문서 제목</title>
</head>

소셜 미리보기

<meta property="og:title" content="오픈그래프 제목">
<meta property="og:description" content="설명">
<meta property="og:type" content="website">
<meta property="og:image" content="https://example.com/cover.png">
<meta name="twitter:card" content="summary_large_image">

시맨틱 HTML과 구조화

시맨틱 태그는 의미를 담아 구조를 선명하게 하고, 검색엔진과 보조기기의 이해를 돕습니다.

핵심 시맨틱 태그

  • header: 상단 영역(로고/내비).
  • nav: 내비게이션 링크 묶음.
  • main: 본문의 고유 핵심 콘텐츠(문서당 1개 권장).
  • section: 주제별 구획(제목 포함 권장).
  • article: 독립적 콘텐츠(카드, 글, 포스트).
  • aside: 보조 정보(사이드바, 관련 링크).
  • footer: 하단 정보(저작권, 링크).

시맨틱 예제

<header>...내비...</header>
<main>
  <article>
    <h1>블로그 글 제목</h1>
    <p>글 본문...</p>
  </article>
  <aside>관련 글</aside>
</main>
<footer>© 2025</footer>
✅
헤딩 구조: <h1>은 문서 핵심 제목, 하위는 논리적으로 h2 → h3 순서로 내려가도록 유지하세요.

폼과 사용자 입력

폼 기본

<form action="/signup" method="post">
  <label for="email">이메일</label>
  <input id="email" name="email" type="email" required placeholder="you@example.com">

  <label for="pw">비밀번호</label>
  <input id="pw" name="pw" type="password" minlength="8" required>

  <button type="submit">가입</button>
</form>
  • label-for: 접근성 향상 및 클릭 영역 확장.
  • required/min/max: 브라우저 기본 검증.
  • type: email, url, number, date 등으로 최적 입력.

고급 입력과 힌트

<fieldset>
  <legend>배송 옵션</legend>
  <label><input type="radio" name="ship" value="standard" checked> 일반</label>
  <label><input type="radio" name="ship" value="express"> 빠른</label>
</fieldset>

<label for="qty">수량</label>
<input id="qty" name="qty" type="number" min="1" max="10" step="1">

<datalist id="cities">
  <option value="Seoul">
  <option value="Busan">
</datalist>
<input list="cities" placeholder="도시 선택">
autocomplete: name, email, country 등 명확한 힌트를 제공하면 사용자 경험이 좋아집니다.

이미지, 오디오, 비디오

이미지 기초와 대체 텍스트

<img src="cat.jpg" alt="창가에 앉아있는 회색 고양이">
  • alt: 이미지의 목적을 설명. 장식적일 땐 빈 값 alt="".
  • width/height: 레이아웃 시프팅 방지 위해 명시 권장.
  • srcset/sizes: 반응형 이미지 제공.
<img
  src="hero-800.jpg" width="800" height="500" alt="도시 야경"
  srcset="hero-400.jpg 400w, hero-800.jpg 800w, hero-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw, 600px">

오디오/비디오

<audio controls>
  <source src="music.mp3" type="audio/mpeg">
  브라우저가 오디오 태그를 지원하지 않습니다.
</audio>

<video controls width="640" height="360" poster="thumb.jpg">
  <source src="movie.mp4" type="video/mp4">
  동영상을 재생할 수 없습니다.
</video>
자막(track): 접근성을 위해 <track kind="subtitles" ...> 제공을 고려하세요.

테이블: 데이터 표현

기본 구조

<table>
  <caption>주간 매출(만원)</caption>
  <thead>
    <tr><th scope="col">요일</th><th scope="col">매출</th></tr>
  </thead>
  <tbody>
    <tr><th scope="row">월</th><td>120</td></tr>
    <tr><th scope="row">화</th><td>140</td></tr>
  </tbody>
  <tfoot>
    <tr><th scope="row">합계</th><td>260</td></tr>
  </tfoot>
</table>

복잡한 테이블 힌트

  • scope: 헤더가 행/열 어느 쪽을 설명하는지.
  • headers/id: 복잡한 매핑 시 명시적 연결.
  • caption: 표의 목적을 설명.
속성의미예
scope헤더의 범위col, row, colgroup, rowgroup
colspan/rowspan셀 병합숫자

웹 접근성 기본

모든 사용자가 접근 가능한 마크업은 선택이 아니라 필수입니다.

스크린리더 친화

  • 대체 텍스트: 의미 있는 이미지는 alt 필수.
  • 폼 레이블: label for로 연결.
  • 색상 대비: 텍스트 대비 4.5:1 이상 권장.
  • 키보드 포커스: 인터랙션 요소는 키보드 접근 가능해야 함.
<a href="#main" class="sr-only">본문 바로가기</a>
<main id="main">...</main>

ARIA는 보완재

  • 원칙: 가능한 시맨틱 태그를 우선, 부족할 때 ARIA 사용.
  • role: 위젯 역할 정의. 예: role="dialog"
  • aria-label/aria-labelledby: 이름 제공.
  • aria-live: 동적 업데이트 공지.
<button aria-expanded="false" aria-controls="menu">메뉴</button>
<nav id="menu" hidden>...</nav>

SEO 기본

콘텐츠 구조와 메타

  • 고유한 title/description: 페이지별로 분리.
  • 헤딩 계층: 논리적 구조로 주제 명확히.
  • 시맨틱 마크업: 의미 있는 태그 활용.
  • 링크 텍스트: "여기 클릭" 대신 의미 있는 텍스트.
<link rel="canonical" href="https://example.com/post/slug">

반응형 레이아웃

뷰포트와 유동 레이아웃

<meta name="viewport" content="width=device-width, initial-scale=1">
          
<div class="container">...</div>
<style>
  .container { max-width: 1200px; margin: 0 auto; padding: 0 16px; }
  .grid { display:grid; gap:16px; grid-template-columns:1fr; }
  @media (min-width: 768px) {
    .grid { grid-template-columns: 1fr 1fr; }
  }
</style>

이미지/미디어 반응형

  • max-width: 100%: 컨테이너 폭에 맞춤.
  • aspect-ratio: 비율 유지.
img { max-width: 100%; height: auto; }
.video { aspect-ratio: 16 / 9; width: 100%; background: #000; }

HTML 관점의 성능 최적화

리소스 힌트

<link rel="preload" as="image" href="/banner.jpg" imagesrcset="/banner-2x.jpg 2x">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link rel="dns-prefetch" href="https://cdn.example.com">
  • preload: 중요한 리소스를 일찍 받기.
  • preconnect: 네트워크 연결 사전 수립.
  • defer: 스크립트 파싱 지연으로 렌더 차단 방지.
<script src="/app.js" defer></script>

레이아웃 안정성

  • 이미지 크기 지정: width/height 명시.
  • 폰트 FOIT/FOUT 완화: font-display: swap.
  • 지연 로딩: loading="lazy" 이미지/iframe.
<img src="photo.jpg" width="1200" height="800" loading="lazy" alt="풍경">

HTML 치트시트

자주 쓰는 태그

  • 텍스트: p, h1-h6, strong, em, blockquote, code
  • 그룹: div, span
  • 시맨틱: header, nav, main, section, article, aside, footer
  • 리스트: ul, ol, li, dl, dt, dd
  • 링크: a, button
  • 미디어: img, picture, source, audio, video

폼

  • 입력: input, textarea, select, option
  • 구조: form, fieldset, legend, label
  • 유효성: required, pattern, min/max, step
  • 자동완성: autocomplete, list/datalist

베스트 프랙티스 체크

  • 문서에 lang, charset, viewport 포함
  • 헤딩 계층 논리적으로 구성
  • 이미지 alt 제공, 크기 지정
  • form 요소에 label 연결
  • 상호작용 요소는 키보드 접근 가능
  • 시맨틱 태그 적극 사용
  • 중요 리소스 preload, 스크립트 defer
